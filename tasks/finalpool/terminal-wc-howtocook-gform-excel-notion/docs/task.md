You are a product development manager at an e-commerce company that sells kitchen appliances. The company wants to launch a new meal kit product line that bundles kitchen appliances with complementary recipe cards. Your job is to analyze the existing product catalog, find recipes that pair well with each appliance, survey customer interest, and build a development tracker.

Start by reading product_strategy.md in your workspace. It describes the meal kit concept, target customer segments, and key goals. Also review pricing_model.csv which contains pricing tier definitions and margin targets, and brand_guidelines.json which specifies branding constraints for the meal kit line.

Query the online store for all products in the "Home Appliances" category. For each product, retrieve the product name, price, regular price, and customer reviews to compute an average rating. There should be 8 products in this category.

Next, query the recipe database for recipes across multiple categories. Search for recipes in at least four different categories including meat dishes, vegetable dishes, staple foods, soups, and seafood. Retrieve at least 12 recipes total, noting each recipe name, category, difficulty level, and key ingredients.

Write a Python script called appliance_recipe_matcher.py in your workspace. The script should match appliances to compatible recipes based on cooking method keywords. Use these matching rules: a blender pairs with recipes that involve mixing, blending, or making drinks and smoothies; a vacuum sealer pairs with recipes involving meat, fish, or food preservation; a cooking pot or steamer pairs with soup, stew, and steamed recipes; a kitchen scale pairs with recipes requiring precise measurements like baking; and an exhaust fan pairs with stir-fry and deep-fry recipes that produce smoke. The script should output a JSON file called appliance_recipe_matches.json containing each appliance name mapped to its list of compatible recipe names. Run this script using command-line tools.

Create a customer interest survey using the online form platform. Title the survey "Meal Kit Interest Survey" and include six questions. First, a multiple choice question asking "How often do you cook at home?" with options Daily, Several times a week, Once a week, Rarely. Second, a multiple choice question asking "What cuisine types interest you most?" with options Chinese, Japanese, Western, Southeast Asian, All types. Third, a checkbox question asking "Which kitchen appliances do you currently own?" with options Blender, Vacuum Sealer, Electric Cooker, Kitchen Scale, Exhaust Fan, None. Fourth, a multiple choice question asking "What is your monthly budget for meal kits?" with options Under 30 dollars, 30 to 60 dollars, 60 to 100 dollars, Over 100 dollars. Fifth, a scale question from 1 to 5 asking "How likely are you to purchase a meal kit bundled with an appliance?" and sixth, a short text question asking "What features would make a meal kit most appealing to you?"

Write a Python script called survey_analyzer.py that reads the survey responses from the database, tallies the most popular answer for each question, and prints a summary. Run this script using command-line tools.

Create an Excel workbook called Meal_Kit_Analysis.xlsx in your workspace with four sheets.

The first sheet should be named Appliance_Catalog and contain columns for product_name, price, avg_rating, and compatible_recipes_count. Include one row for each of the 8 Home Appliances products. The price should come from the store catalog. The avg_rating should be the average of all customer review ratings for that product rounded to two decimal places. The compatible_recipes_count should be the number of recipes matched to that appliance by your matching script.

The second sheet should be named Recipe_Matches and contain columns for recipe_name, matched_appliance, category, and difficulty. Include one row for each appliance-recipe pairing found by the matching script. If a recipe matches multiple appliances, include it once for each match.

The third sheet should be named Survey_Results and contain columns for question, top_answer, and response_count. Include one row for each of the six survey questions showing the most frequently selected answer and how many respondents chose it.

The fourth sheet should be named Product_Roadmap and contain columns for kit_name, included_appliance, recipe_count, estimated_price, and priority. Create 5 meal kit concepts, one for each major appliance type (blender, vacuum sealer, cooking pot, kitchen scale, exhaust fan). The kit_name should be a descriptive name like "Smoothie Starter Kit" or "Stir-Fry Master Kit". The recipe_count is how many recipes matched that appliance. The estimated_price should be the appliance price plus 15 dollars for recipe cards and packaging. The priority column should be "High" if the kit has more than 3 compatible recipes, otherwise "Medium".

Create a database in the team knowledge base called "Meal Kit Development Tracker" with properties for Title (title type), Status (select type with options Planning, In Development, Ready for Launch, Launched), Priority (select type with options High, Medium, Low), Launch_Date (date type), and Estimated_Revenue (number type). Add one page for each of the 5 meal kit concepts from the Product_Roadmap sheet. Set the Status to "Planning" for all kits. Set Priority to match the priority from the roadmap. Set Estimated_Revenue to the estimated_price multiplied by 100 units.
