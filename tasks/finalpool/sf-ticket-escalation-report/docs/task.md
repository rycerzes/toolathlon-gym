The support manager needs a weekly escalation analysis report that identifies tickets at risk of breaching service level agreements and tracks customer satisfaction issues across the support organization. The current escalation rules are maintained as a JSON configuration at http://localhost:30210/api/escalation_rules.json which defines the SLA thresholds by priority level, the customer satisfaction threshold below which tickets are flagged, and the response time warning percentage that indicates when a ticket is approaching its SLA limit.

Start by reading the escalation protocol document and the team contacts file in the workspace to understand the current procedures and distribution list. Then fetch the escalation rules from the API endpoint to get the specific thresholds for analysis. Query the support ticket data warehouse to analyze all tickets against these rules.

Using the response time warning percentage from the escalation rules, identify tickets where the response time exceeds that fraction of the SLA hours as at-risk tickets. Also identify all tickets where customer satisfaction falls below the satisfaction threshold defined in the rules. Break down these findings by priority level, by issue type, and by the ticket reporter.

Create an Excel file called Escalation_Report.xlsx in the workspace with five sheets. The first sheet named At Risk Tickets should have columns Priority, Total_Tickets, At_Risk_Count showing tickets where response time exceeds the warning threshold fraction of SLA hours, At_Risk_Pct, Avg_Response_Hours, and SLA_Hours, sorted by priority. The second sheet named Low Satisfaction should have columns Priority, Issue_Type, Low_CSAT_Count where satisfaction is below the threshold, Total_Count, Low_CSAT_Pct, and Avg_CSAT, sorted by Low_CSAT_Count descending. The third sheet named Priority Summary should have columns Priority, Total_Tickets, Low_CSAT_Tickets, Low_CSAT_Pct, Avg_Response_Hours, Avg_CSAT, and SLA_Hours, sorted by priority. The fourth sheet named Reporter Analysis should have columns Reporter, Total_Tickets, Low_CSAT_Count, Low_CSAT_Pct, Avg_Response_Hours, and Avg_CSAT, sorted by reporter name. The fifth sheet named Summary should have Metric and Value columns listing Total_Tickets, Low_Satisfaction_Tickets, Low_Satisfaction_Pct, At_Risk_Tickets, At_Risk_Pct, Avg_Response_Hours, Avg_Customer_Satisfaction, Worst_Issue_Type by average satisfaction, Worst_Issue_Avg_CSAT, Highest_Risk_Priority by at-risk percentage, and Highest_Risk_Pct.

Send an email to the escalation group address from the team contacts file with the subject Weekly Escalation Analysis Report. The email body should summarize the total ticket count, the number and percentage of low satisfaction tickets, the number of at-risk tickets, the worst performing issue type, and a note that the full Excel report is available for review.

Create a Notion page titled Support Escalation Analysis with content summarizing the key findings and listing at least three specific action items based on the analysis, such as focusing attention on the issue types with lowest satisfaction or reviewing the priority level with the highest risk percentage.
