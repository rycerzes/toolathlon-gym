You are a sales VP preparing a research-backed quarterly strategy presentation. Your goal is to combine internal sales performance data with academic research on sales methodologies to build a compelling, evidence-based strategy review.

Start by reading the strategy_brief.txt file in your workspace, which outlines the strategic goals and revenue targets for next quarter. Also read the sales_team_roster.csv file to understand the regional sales managers and their individual targets. Finally, review historical_benchmarks.json which contains the last four quarters of performance data for context.

From the enterprise data warehouse, query the sales database to get regional performance data. For each region, retrieve the total revenue, order count, and average order value. Also pull customer data grouped by segment including the customer count, total spend, and average lifetime value per segment. Then query product performance to identify the top 5 and bottom 5 products by revenue, including their category information.

Write and run a Python script called analyze_sales_gaps.py in your workspace. The script should compute a year-over-year revenue change estimate for each region by comparing current totals against the quarterly benchmarks from the historical data file. It should identify which regions fall below the median revenue as underperforming, and tag those as "Priority Focus" regions. The script should output a file called sales_gaps.json containing the region performance data along with the priority tags.

Search the research paper database for academic papers related to "sales optimization" and "customer segmentation strategies". You should find several papers on these topics. Filter to only those papers whose abstracts discuss actionable sales strategies, regional performance improvement, or customer lifetime value optimization. Ignore any papers focused on unrelated domains like healthcare or pure machine learning theory.

Write and run a second Python script called match_recommendations.py. This script should read the sales_gaps.json file and the relevant paper information, then produce research_recommendations.json. For each underperforming region or segment gap identified, the script should match at least one research-backed recommendation from the paper abstracts, noting the paper title, the key finding, and the expected impact area.

Check the shared calendar for existing events in the next two weeks to find available time. Then create a calendar event called "Q1 Sales Strategy Review" scheduled in the first available 90-minute slot that does not conflict with existing events. The event should be during business hours between 9 AM and 5 PM on a weekday. Add a description mentioning that the meeting covers regional performance review and research-backed action items.

Create an Excel workbook called Sales_Strategy_Analysis.xlsx in your workspace with four sheets. The first sheet named Regional_Performance should have columns for region, revenue, order_count, avg_order_value, yoy_change_pct, and priority_tag. Include one row per region with the priority_tag column showing "Priority Focus" for regions below the median revenue and "On Track" for the rest. The second sheet named Segment_Analysis should have columns for segment, customer_count, total_spend, and avg_ltv. Include one row per customer segment. The third sheet named Research_Insights should have columns for paper_title, key_finding, applicable_region, and expected_impact. Include one row per research recommendation matched to an underperforming area. The fourth sheet named Action_Items should have columns for action, region, timeline, owner, and research_basis. Include at least one action item per Priority Focus region, with the owner drawn from the sales_team_roster.csv and the research_basis referencing a specific paper title.

Create a PowerPoint presentation called Sales_Strategy_Presentation.pptx in your workspace with six slides. The first slide should be a title slide with "Q1 Sales Strategy Review" as the title and the current date as subtitle. The second slide should present the regional performance overview with key revenue figures. The third slide should show the segment analysis with customer counts and spend data. The fourth slide should summarize the research findings and how they apply to identified gaps. The fifth slide should present the action plan with specific initiatives tied to research. The sixth slide should show a timeline of key milestones for the quarter.

Finally, write and run a Python script called generate_summary.py that reads the sales_gaps.json and research_recommendations.json files, then produces an executive_summary.txt file in your workspace. The summary should be a concise paragraph highlighting the total revenue across all regions, the number of Priority Focus regions, and the number of research-backed recommendations generated.
