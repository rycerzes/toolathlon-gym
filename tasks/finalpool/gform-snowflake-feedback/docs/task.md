The support operations team needs to create a customer feedback program based on actual support ticket data from the company data warehouse. The data warehouse contains a support center database with ticket records including issue types, priorities, response times, and customer satisfaction scores.

Start by querying the data warehouse to identify the top five issue types by total ticket volume from the SUPPORT_CENTER.PUBLIC.TICKETS table. For each of those five issue types, note the ticket count, average response time in hours, and average customer satisfaction score.

Next, create a Google Form titled "Customer Feedback Survey" with a description of "Help us improve our support services by sharing your feedback on recent issues." The form should contain at least five questions. Include a question asking which issue type the customer experienced, presented as a multiple choice question with the top five issue types as options. Include a question asking the customer to rate their overall satisfaction on a scale from 1 to 5. Include a question asking what could be improved about the support experience, as a free text response. Include a question asking how likely the customer is to recommend the support service to others on a scale of 1 to 10, as a free text or short answer question. Include a question asking for any additional comments or suggestions, as a free text response.

Then create an Excel workbook called Support_Feedback_Analysis.xlsx in the workspace with three sheets. The first sheet should be named "Ticket Analysis" and contain columns Issue_Type, Ticket_Count, Avg_Response_Time, and Avg_Satisfaction. Populate it with the top five issue types sorted by ticket count in descending order, where Avg_Response_Time is rounded to two decimal places and Avg_Satisfaction is rounded to two decimal places. The second sheet should be named "Priority Breakdown" and contain columns Priority, Count, Pct_of_Total, and Avg_Response_Time. Populate it with one row per priority level sorted alphabetically by priority name, where Pct_of_Total is the percentage of total tickets rounded to two decimal places and Avg_Response_Time is rounded to two decimal places. The third sheet should be named "Improvement Plan" and contain columns Issue_Type, Current_Satisfaction, Target_Satisfaction, and Action_Item. Populate it with the top five issue types sorted by ticket count in descending order. Current_Satisfaction should match the average satisfaction from the Ticket Analysis sheet. Target_Satisfaction should be the current satisfaction plus 0.5 rounded to two decimal places. Action_Item should contain a brief one-sentence description of a specific improvement action for that issue type.

Create a Word document called Support_Improvement_Report.docx in the workspace. The document should include a title "Support Service Improvement Report", an executive summary section that mentions the total number of tickets analyzed and the overall average satisfaction score, a key findings section that discusses the top issue types and their satisfaction scores, and an improvement recommendations section that outlines specific actions for each of the top five issue types.

Finally, send an email to support-team@company.com with the subject "Support Improvement Report - Action Required". The email body should include a summary of the key findings from the analysis, mentioning the total number of tickets, the top issue type by volume, and the overall average satisfaction score.
