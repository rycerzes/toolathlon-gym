I am a department chair preparing a self-study report for our upcoming program accreditation review. There is a PDF document in the workspace called accreditation_standards.pdf that outlines eight accreditation criteria our program must meet, each with a specific minimum threshold. Please start by reading this PDF to understand all the criteria and their requirements.

Next, pull course data from our learning management system. Get the full list of courses offered in our program, and for each course retrieve the enrollment count, average submission grade, and average quiz score. Focus especially on the two Creative Computing and Culture course sections (courses 7 and 8) since those are our flagship offerings and the accreditation reviewers always look at them closely.

There is also a file called previous_review.csv in the workspace with results from our last accreditation cycle, and a file called faculty_credentials.json listing our faculty qualifications. Use these as reference when assessing compliance.

Write and run a Python script called compute_metrics.py in the workspace that reads the course data you gathered and computes program-level metrics. The script should calculate the overall average GPA across all courses by averaging the per-course average grades on a 0-100 scale, the retention proxy which is total enrollment divided by total submission count across all courses, and an assessment diversity score computed as the number of courses that have both assignments and quizzes divided by the total number of courses. The script should output its results to program_metrics.json in the workspace.

Then write and run a second script called evaluate_compliance.py that reads the accreditation criteria from accreditation_standards.pdf (you can hardcode the thresholds you extracted earlier) and the program_metrics.json file, and evaluates each of the eight criteria. For each criterion it should determine a status of Compliant if the metric meets or exceeds the threshold, Partial if within 10 percent of the threshold, or Non-Compliant if below. Output the results to compliance_assessment.json.

Create an Excel workbook in the workspace called Accreditation_Self_Study.xlsx with four sheets. The first sheet Course_Data should have columns Course_Name, Enrollment, Avg_Grade, Quiz_Avg, and Assignment_Count, with one row per course sorted alphabetically. The second sheet Accreditation_Matrix should have columns Criterion, Threshold, Actual_Value, and Status with one row per accreditation criterion. The third sheet Gap_Analysis should have columns Criterion, Gap_Description, Remediation_Action, and Timeline, with rows only for criteria that are Partial or Non-Compliant. The fourth sheet Resource_Needs should have columns Resource, Justification, Estimated_Cost, and Priority, listing at least three resources needed to address compliance gaps such as additional faculty hires, assessment software, or curriculum redesign consulting.

Create a database in the team knowledge base called "Accreditation Action Items" with properties Criterion as title, Status as a select field with options Compliant and Partial and Non-Compliant, Owner as rich text, Due_Date as date, and Evidence as rich text. Add one entry for each of the eight accreditation criteria. Set the status based on the compliance assessment results, assign owners from the faculty listed in faculty_credentials.json, set due dates spread across the next three months, and note what evidence is needed for each.

Check the shared calendar for any existing accreditation-related events. Then create three new calendar events. First an event called "Evidence Collection Deadline" scheduled for 30 days from now. Second an event called "Draft Report Due" scheduled for 60 days from now. Third an event called "Final Submission" scheduled for 90 days from now. Each event should include a description mentioning the accreditation self-study.

Finally, write and run a script called generate_summary.py that reads the compliance assessment results and produces a plain text file called accreditation_summary.txt in the workspace. This summary should list each criterion with its status, highlight any gaps found, and state the overall compliance percentage.
