The academic affairs office wants to evaluate whether the current course curricula align with recent research trends. There is a PDF file called Curriculum_Review_Guidelines.pdf in the workspace that describes the review methodology and scoring criteria. There is also a JSON file called review_criteria.json with relevance scoring thresholds. Please review both files first.

Start by pulling course information from the learning management system. Get the list of all courses along with their modules. Focus on the unique course subjects (for example, Applied Analytics and Algorithms, Data-Driven Design, Foundations of Finance, etc.) and identify the key topic areas covered by each subject based on module names and course descriptions.

Next, search the research paper archive for recent publications related to the course topics. Search for papers about data analytics, machine learning, algorithm design, data-driven approaches, financial analysis, and computational methods. Retrieve at least 5 relevant papers and note their titles, categories, and publication dates.

Using the command line, write and run a Python script that performs a curriculum gap analysis. The script should map each unique course subject to the research papers that are relevant to it, assign a relevance score between 0 and 10 for each paper-course pair based on keyword overlap between the paper summary and the course subject area, and identify gaps where current courses do not cover emerging research topics. Save the analysis output to a file called curriculum_gap_output.txt in the workspace.

Create a Word document called Curriculum_Enhancement_Proposal.docx in the workspace. The document should have a title "Curriculum Enhancement Proposal", an introduction explaining the purpose of the review, a section for each course subject with specific recommendations based on related research papers, and a conclusion summarizing the most critical curriculum gaps.

Create an Excel file called Curriculum_Gap_Analysis.xlsx in the workspace with three sheets. The first sheet should be called "Course_Topics" with columns Course_Subject and Current_Topics listing each unique course subject and a summary of its module topics. The second sheet should be called "Research_Frontiers" with columns Paper_Title, Primary_Category, and Relevance_Score listing each research paper found with a general relevance score from 1 to 10. The third sheet should be called "Gap_Matrix" with columns Course_Subject, Paper_Title, Relevance_Score, and Gap_Type where Gap_Type is either "Covered" if the relevance score is 7 or above, "Partial" if between 4 and 6, or "Gap" if below 4.

Finally, create a database in the knowledge base titled "Curriculum Review Tracker" with properties for Course Subject (title), Review Status (select with options Not Started, In Progress, and Completed), Priority (select with options High, Medium, and Low), and Notes (rich text). Add one entry for each unique course subject with the Review Status set to Not Started and Priority based on how many research gaps were identified for that subject.
