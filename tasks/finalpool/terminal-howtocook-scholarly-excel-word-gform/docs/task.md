You are a corporate wellness coordinator tasked with designing an evidence-based workplace nutrition program. Your goal is to combine practical recipe data with published nutritional science research to create a scientifically grounded meal program for employees.

Start by reading the program_requirements.md file in your workspace, which describes the wellness program goals, target population size, and budget constraints. Also read the dietary_restrictions.csv file that lists common employee dietary restrictions along with the number of employees affected by each restriction. Finally, review nutritional_targets.json which specifies daily nutritional targets including calorie ranges, minimum protein intake, minimum fiber intake, and maximum sodium limits.

Query the recipe database to retrieve recipes from multiple categories. You need at least three meat dishes, three vegetable dishes, two staple foods, and two soups. For each recipe, note the name, category, difficulty level, and ingredients.

Write a Python script called categorize_recipes.py in your workspace that takes the retrieved recipes and categorizes each one by dietary compatibility. A recipe is vegetarian if it contains no meat or seafood ingredients. A recipe is low-sodium if it does not list soy sauce, salt-heavy seasonings, or pickled ingredients as primary components. A recipe is high-protein if it is a meat dish, contains eggs, or contains tofu. The script should output a file called categorized_recipes.json containing each recipe with its name, category, dietary tags as a list, estimated prep time in minutes (use 15 for difficulty 1, 25 for difficulty 2, 35 for difficulty 3, 45 for difficulty 4, 60 for difficulty 5), and a difficulty score.

Execute the categorize_recipes.py script using the command line.

Search the research paper database for studies related to "workplace nutrition program" and "dietary intervention employee health". You should find papers about workplace dietary interventions, nutrition program effectiveness, and employee health outcomes. From the results, identify the papers that are specifically about workplace or employee nutrition programs and ignore any papers about unrelated topics like sports nutrition or agricultural policy.

Write a Python script called analyze_research.py that processes the relevant research papers. For each relevant paper, extract the title, publication year, venue, citation count, and a brief summary of the key finding from the abstract. Assign each paper a relevance score from 1 to 5 based on how directly it addresses workplace nutrition programs (5 being most relevant). The script should output research_findings.json with these fields for each paper.

Execute analyze_research.py via the command line.

Write a Python script called build_menus.py that combines the categorized recipes with the research findings to create evidence-based weekly menus. The script should create five daily menus for Monday through Friday where each day has a breakfast suggestion (use a simple default like "Oatmeal with fruit" or "Whole grain toast with eggs"), a lunch recipe selected from the categorized recipes, and a dinner recipe selected from the categorized recipes. No two consecutive days should have lunch from the same recipe category. Each daily menu should include an estimated calorie count (breakfast 400 calories, lunch based on category where meat is 650 calories and vegetable is 350 calories and staple is 500 calories and soup is 300 calories, dinner similarly). Calculate a dietary compliance percentage for each day as the percentage of the four nutritional targets (calories between 1800 and 2200, protein above 50 grams, fiber above 25 grams, sodium below 2300 milligrams) that the menu is estimated to meet, using reasonable estimates. The script should output evidence_based_menus.json.

Execute build_menus.py via the command line.

Create an online survey form titled "Employee Dietary Preferences Survey" with the following questions. First, a checkbox question titled "Which dietary restrictions apply to you?" with options Vegetarian, Vegan, Gluten-Free, Low-Sodium, Dairy-Free, Nut Allergy, and No Restrictions. Second, a multiple choice question titled "Which meal of the day would you most like the program to focus on?" with options Breakfast, Lunch, Dinner, and Snacks. Third, a multiple choice question titled "How much would you be willing to spend per meal through a subsidized program?" with options Under $5, $5-$8, $8-$12, and Over $12. Fourth, a yes/no multiple choice question titled "Would you be interested in attending cooking workshops?" with options Yes and No.

Create an Excel workbook called Nutrition_Program_Analysis.xlsx in your workspace with four sheets.

The first sheet should be named Recipe_Database and contain columns recipe_name, category, dietary_tags, prep_time_min, difficulty, and evidence_score. Include one row for each recipe you retrieved from the recipe database. The evidence_score column should contain a value from 1 to 5 indicating how well the recipe aligns with research recommendations (give higher scores to recipes matching categories that research papers recommend, default 3 for recipes without specific research backing).

The second sheet should be named Research_Summary and contain columns paper_title, pub_year, venue, finding, confidence_level, and applicable_dietary_type. Include one row for each relevant research paper you found. The confidence_level should be High if citation count is above 100, Medium if between 20 and 100, and Low otherwise. The applicable_dietary_type should describe what dietary category the finding most applies to.

The third sheet should be named Weekly_Menu and contain columns day, breakfast, lunch, dinner, est_calories, and dietary_compliance_pct. Include five rows for Monday through Friday with the menus from your build_menus.py output.

The fourth sheet should be named Program_Budget and contain columns item, cost_per_person, participants, and total_cost. Include rows for Recipe ingredient costs per week with cost 35 dollars and 200 participants, Cooking workshop monthly with cost 15 and 50 participants, Nutritional consultation quarterly with cost 25 and 200 participants, Survey platform annual with cost 2 and 200 participants, and Program administration monthly with cost 10 and 200 participants. The total_cost column should be cost_per_person multiplied by participants.

Create a Word document called Nutrition_Program_Proposal.docx in your workspace. The document should have the title "Evidence-Based Workplace Nutrition Program Proposal" followed by these sections. An Executive Summary section describing the program purpose, target of 200 employees, and evidence-based approach. A Research Foundation section summarizing the key findings from the research papers and how they inform the program design. A Recipe Selection Methodology section explaining how recipes were categorized by dietary compatibility and matched with research recommendations. A Proposed Weekly Menus section describing the five-day menu plan with calorie estimates and compliance percentages. A Survey Plan section describing the employee dietary preferences survey and how responses will be used to refine menus. A Budget Overview section summarizing the program costs from the Excel budget sheet. A Timeline section proposing a three-phase rollout with Phase 1 Survey and Planning in month one, Phase 2 Pilot Program in months two through three, and Phase 3 Full Rollout in month four onward.

Finally, write a validation script called validate_menus.py that reads the evidence_based_menus.json file and checks that each daily menu has all three meals specified, that estimated calories are within a reasonable range of 1400 to 2800, that no two consecutive days have the same lunch category, and that dietary compliance percentages are between 0 and 100. Execute this script and confirm all validations pass.
