The student affairs office is conducting a pilot study on whether nutrition awareness correlates with academic performance. The study focuses on two Environmental Economics and Ethics courses (course IDs 13 and 14) from the learning management system.

Start by reading the study_protocol.md file in the workspace to understand the research methodology and IRB requirements. Also review the nutritional_guidelines.csv and campus_dining_budget.json files for context on daily nutritional requirements and budget constraints.

Query the learning management system for enrollment and grade data from both courses. For each course, retrieve assignments with their point values and submission counts. Also gather student-level submission patterns, specifically looking at which students have late submissions and their average scores.

Query the cooking recipe database for healthy recipe categories, focusing on breakfast and lunch options. Retrieve all categories along with recipe counts per category.

Write a Python script called categorize_students.py that reads the submission data and categorizes students into three engagement tiers. Students with zero late submissions are "High" engagement, students with up to 25% late submission rate are "Medium" engagement, and students above 25% late rate are "Low" engagement. The script should compute for each tier the student count, average score, and late submission rate. Run this script via the terminal and save the output as student_engagement.json containing an array of objects with fields tier, student_count, avg_score, and late_submission_rate.

Write a second Python script called design_meal_plans.py that reads the student_engagement.json and the nutritional_guidelines.csv, then designs meal plan recommendations for each tier. High engagement students get a "Control" plan with no special meals. Medium engagement students get a "Partial" plan with breakfast recommendations only. Low engagement students get a "Full Meal Plan" with both breakfast and lunch recommendations. The script should pull breakfast and lunch recipe categories from the cooking database results and estimate a daily cost based on the campus_dining_budget.json constraints. Run this script and save the output as meal_recommendations.json.

Create a cloud spreadsheet titled "Nutrition Study Data" with two sheets. The first sheet called "Student_Engagement" should have columns Tier, Student_Count, Avg_Score, and Late_Submission_Rate, with one row per tier (High, Medium, Low) sorted by tier name. The second sheet called "Meal_Plans" should have columns Tier, Intervention_Type, Recommended_Meals, and Estimated_Daily_Cost. High tier gets "Control" intervention with "None" for meals and cost 0. Medium tier gets "Partial" intervention with breakfast recipes listed and estimated cost from budget. Low tier gets "Full Meal Plan" intervention with breakfast and lunch recipes listed and estimated cost from budget.

Create a knowledge base database titled "Wellness Pilot Program" with properties Student_Group (title), Intervention_Type (select with options Full Meal Plan, Partial, Control), Enrollment (number), Start_Date (date), and Status (select with options Planning, Active, Completed). Add three entries, one for each engagement tier. The High tier group has Control intervention with the count of high-engagement students, start date 2026-04-01, status Planning. The Medium tier group has Partial intervention with the count of medium-engagement students, same start date and status. The Low tier group has Full Meal Plan intervention with the count of low-engagement students, same start date and status.

Send an email to student_affairs@university.edu with subject "Nutrition-Academic Performance Study: Pilot Proposal" summarizing the study findings. The body should mention the three engagement tiers with their student counts and average scores, the proposed intervention types, and request approval to proceed with the pilot starting April 2026.

Send a second email to dining_services@university.edu with subject "Meal Plan Recommendations for Wellness Pilot" describing the meal plan requirements. Include the number of students needing full and partial meal plans, the recommended breakfast and lunch categories from the recipe database, and budget estimates per tier.

Finally, write a Python summary script called generate_summary.py that reads student_engagement.json and meal_recommendations.json, then produces a comprehensive text report. Run this script and save the output as nutrition_study_summary.txt in the workspace. Also save a local Excel backup called Nutrition_Academic_Study.xlsx with two sheets matching the cloud spreadsheet structure: "Student_Engagement" with columns Tier, Student_Count, Avg_Score, Late_Submission_Rate, and "Meal_Plans" with columns Tier, Intervention_Type, Recommended_Meals, Estimated_Daily_Cost.
