You are a support operations manager building an SLA compliance dashboard that compares your internal support performance with industry benchmarks. Your goal is to analyze support ticket data, fetch external benchmark data, compute compliance rates, and publish findings across multiple platforms.

Start by reading the SLA_Framework.pdf in your workspace, which defines internal SLA targets and the methodology for computing compliance rates. Also read the benchmark_config.json file, which provides the URL for fetching industry benchmark data and the parameters for comparison.

Fetch the industry SLA benchmarks from the dashboard page at http://localhost:30185. The page contains a table of industry-standard metrics including average response time, customer satisfaction scores, first contact resolution rate, and overall SLA compliance percentage. Record these values for comparison with your internal data.

From the enterprise data warehouse, query the support center database. Retrieve ticket data grouped by priority level, including the total number of tickets, the average response time in hours, and the average customer satisfaction score for each priority. Also query the SLA policies table to get the response target hours for each priority level. Retrieve agent performance data showing each agent's name, team assignment, the number of tickets they handled, and their average response time.

Using command-line tools, write and execute a Python script called sla_analyzer.py in your workspace. The script should compute SLA compliance rates for each priority level. A ticket is considered SLA-compliant if its average response time is at or below the SLA target for that priority. Calculate the compliance rate as a percentage. Then compare the internal overall metrics against the industry benchmarks fetched from the dashboard, computing the gap for each metric and flagging whether the internal value meets or falls short of the benchmark.

Create an Excel workbook called SLA_Compliance_Report.xlsx in your workspace with four sheets. The first sheet named Ticket_Summary should contain columns for priority, ticket count, average response hours, average satisfaction rating, SLA target hours, and compliance rate as a percentage. Include one row for each priority level. The second sheet named Benchmark_Comparison should contain columns for metric name, internal value, industry benchmark value, gap, and status where status is "Meets" if the internal value is as good as or better than the benchmark and "Below" otherwise. Include rows for average response time, customer satisfaction, and SLA compliance rate. The third sheet named Agent_Performance should contain columns for agent name, team, tickets handled, average response time, and whether they meet SLA targets. Include one row per agent. The fourth sheet named Monthly_Trend should contain columns for month, total tickets, and an estimated compliance percentage. Include rows for at least three recent months with plausible trending data based on the overall compliance patterns you observe.

Create a knowledge base database called "SLA Compliance Dashboard" with properties for Metric Name as a title field, Value as a number field, Benchmark as a number field, Status as a select field with options Meets and Below, and Updated as a date field. Add one page entry for each of the key comparison metrics from your benchmark analysis.

Schedule monthly SLA review meetings on the shared calendar. Create three events on the first Monday of April, May, and June 2026 at 10 AM, each lasting one and a half hours. The event summaries should include "Monthly SLA Review" and the month name. Add a description mentioning that the meeting covers SLA compliance review and action items.
