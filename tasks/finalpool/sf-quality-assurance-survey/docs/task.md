Our quality assurance team needs a comprehensive assessment of support ticket handling performance. The first step is to visit our internal quality standards portal at http://localhost:30213 to review the current target metrics for response times, customer satisfaction, and resolution rates. The portal has two pages, the main page showing target numbers and a standards page with best practices details.

After reviewing the quality targets from the portal, pull all support ticket data from our database. We track tickets with fields like priority level, response time, customer satisfaction rating, issue type, and resolution status. We also have a separate table listing our support agents with their team assignments and skill levels. Analyze each agent by computing their average response time across all their tickets, their average customer satisfaction score, and their overall resolution rate as a percentage. Note that in our system the person recorded on each ticket is the one who handled it. Compare each agent metric against the portal targets and identify which areas fall below standard for each agent.

Create an Excel workbook called QA_Assessment.xlsx in your workspace. The first sheet should be named "Agent Scorecard" with columns for the agent name, their average response time in hours rounded to two decimal places, their average customer satisfaction rounded to two decimals, their resolution rate as a percentage rounded to one decimal, and a text field listing all the target areas where the agent falls below the published standards (for example "Response Time, Satisfaction" if both are below target). Sort the rows alphabetically by agent name.

The second sheet should be called "Issue Type Analysis" and should break down performance by issue type. Include columns for the issue type name, total ticket count for that type, average response time in hours rounded to two decimals, and average satisfaction score rounded to two decimals. Sort alphabetically by issue type.

The third sheet should be named "Summary" with two columns, Metric and Value. Include the following summary metrics: Total_Tickets (total ticket count across all data), Overall_Avg_Response_Time (the overall average response time rounded to two decimals), Overall_Avg_Satisfaction (overall average satisfaction rounded to two decimals), Overall_Resolution_Rate (overall percentage rounded to one decimal), Agents_Below_Satisfaction_Target (count of agents whose average satisfaction is below the portal target), and Agents_Below_Response_Target (count of agents whose overall average response time exceeds a reasonable benchmark, specifically those above the medium priority response target from the portal).

Also review the QA checklist markdown file provided in your workspace for context on scoring methodology, and check the team roster CSV for the list of agents. Note that the CSV may have some data quality issues like extra blank rows or inconsistent formatting that you should be aware of.

Finally, create a Google Form titled "QA Self Assessment Survey" that agents can use to self-evaluate their performance on the identified weak areas. The form should contain five questions. The first question should ask for the agent name as a short text field and should be required. The second question should ask which team they belong to, as a multiple choice question with options matching the teams from the agents table, and should be required. The third question should ask them to rate their own response time performance on a scale, using a multiple choice with options "Exceeds Target", "Meets Target", "Below Target", and "Needs Improvement", and should be required. The fourth question should ask about their biggest challenge in meeting quality targets, as a paragraph text field, and should be required. The fifth question should ask for suggested improvements to the QA process, as a paragraph text field, and should not be required.
